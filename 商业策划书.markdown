# InsightNote 商业计划书 (Business Plan)

> **Slogan:** We read the 100-page institutional reports so you don't have to.  
> **核心定位:** 面向个人投资者的“专业研报过滤器”与“金融逻辑翻译官”。  
> **日期:** 2025年12月

---

## 1. 执行摘要 (Executive Summary)

**InsightNote** 是一个结合了“深度内容订阅”与“轻量级金融工具”的垂直资讯平台。

我们致力于解决普通投资者在面对海量金融信息时存在的“看不懂（专业壁垒）”、“看不完（信息过载）”和“用不上（缺乏实操性）”三大痛点。通过“人话翻译”顶级机构研报，并辅以可视化的逻辑推演工具（如蝴蝶效应图谱），帮助中产阶级、互联网从业者及金融爱好者建立机构视角的投资逻辑，实现低成本的专业知识获取与资产配置优化。

---

## 2. 市场痛点与机会 (Market & Problem)

### 2.1 现有痛点
1.  **信息噪音与情绪化：** 市场充斥着碎片化新闻和情绪化“小作文”，缺乏深度逻辑支撑。
2.  **专业门槛过高：** 机构研报（如高盛、大摩）篇幅长、术语多，普通人难以消化。
3.  **工具与内容割裂：** 用户在阅读资讯和进行投资决策之间存在断层，缺乏将“宏观逻辑”转化为“选股/配置思路”的中间层产品。

### 2.2 目标受众 (Target Audience)

| 用户分层 | 典型画像 | 核心需求 |
| :--- | :--- | :--- |
| **一级 (Core)** | 金融从业者、资深股民 | 跨行业知识补充，寻找非共识性的 Alpha 机会。 |
| **二级 (Main)** | 互联网/科技大厂员工 | 相信技术与逻辑，有理财刚需，愿意为节省时间付费。 |
| **三级 (Mass)** | 寻求资产配置的个人 | 听懂宏观政策，避免盲目投资，需要保姆级解读。 |

## 3. 产品与服务 (Products & Services)

采用 **“Newsletter（内容） + Web Tools（工具）”** 的混合模式。

### 3.1 核心内容板块 (Content)

* **每周深度解读 (The Deep Dive)**
    * **选材：** 顶级投行、VC 或智库的重磅报告。
    * **结构：**
        * `TL;DR (1min)`：核心结论速览。
        * `Logic Unpacked`：去除术语，商业逻辑拆解。
        * `InsightNote Take`：结合本土市场的独家点评。
* **Fintech Stack (金融科技工具箱)**
    * 评测并教授如何使用提高投资效率的工具（如 TradingView 高级脚本、链上数据工具、AI 财报分析 Prompt）。
* **大咖访谈 (Expert Corner)**
    * 采访量化交易员、基金经理等，挖掘其决策模型与信息源。

### 3.2 差异化工具矩阵 (The Tool Aspect)

* **🦋 蝴蝶效应图谱 (Butterfly Effect Map) —— [旗舰功能]**
    * **定义：** 动态思维导图工具，展示事件传导链路。
    * **场景：** 用户点击“美联储降息” -> 图谱展开：美元指数下跌 -> 大宗商品上涨 -> 新兴市场流动性改善。
    * **价值：** 可视化“牵一发而动全身”的投资逻辑。
* **🔍 研报筛选器 (Report Database)**
    * 结构化的知识库。支持按“看多/看空”、“目标价”、“核心风险”筛选历史解读文章。
* **📰 新闻联播助手 (Macro Brief)**
    * 每日提取 3-5 个政策关键词，翻译为市场语言（利好/利空板块）。

## 4. 商业模式与盈利 (Business Model)

采用 **Freemium (免费+增值)** 模式，主打高性价比，通过规模效应盈利。

### 4.1 价格策略

* **年费：** **$59 - $79 / 年** (约 ¥399 - ¥569) —— *主推，极具竞争力*
* **月费：** **$9.9 / 月** —— *锚定价格*

### 4.2 权益分层 (Access Control)

| 权益项目 | 免费用户 (Free) | 订阅用户 (Pro) |
| :--- | :--- | :--- |
| **深度研报 (Deep Dive)** | 仅限查看 **1个月前** 的归档 | **实时**查看最新深度解读 |
| **简报 (TL;DR)** | ✅ 每周接收 | ✅ 每周接收 |
| **工具权限** | ❌ 仅限基础预览 | ✅ **蝴蝶效应图谱、筛选器**全功能 |
| **历史归档** | ✅ 开放（作为SEO流量池） | ✅ 无限制检索与下载 |
| **社群/客服** | ❌ | ✅ 优先解答 |

## 5. 运营与增长策略 (Growth Strategy)

### 5.1 冷启动 (0 - 1,000 付费用户)
* **Infographics 引流：** 制作“一张图看懂XX研报”发布于 Twitter/即刻/小红书，将流量引导至 Newsletter 订阅。
* **工具诱饵：** 开放“蝴蝶效应图谱”的一个热门案例（如“地缘政治对油价影响”）供免费试玩，激发用户对工具的好奇心。

### 5.2 扩张期
* **SEO 策略：** 针对长尾关键词（如“SaaS 估值模型”、“2025 新能源出海”）优化内容，截获搜索流量。
* **Affiliate 计划：** 老用户推荐新用户，双方获赠 1 个月会员时长。

## 6. 技术实现规划 (Technical Roadmap)

### 阶段一：MVP (最小可行性产品)
* **目标：** 验证内容付费意愿。
* **栈：** `Substack` / `Ghost` (内容托管) + `Notion` (简易数据库)。
* **成本：** 极低，主要是时间成本。

### 阶段二：独立站与工具化
* **目标：** 上线工具，建立护城河。
* **前端：** `Next.js` + `Tailwind CSS` (高性能、极简设计)。
* **后端：** `Supabase` (数据库与鉴权)。
* **可视化：** `React Flow` 或 `D3.js` (实现蝴蝶效应图谱的交互)。
* **支付：** `Stripe` (国际) + 微信支付接入。

## 8. 总结 (Conclusion)

InsightNote 的核心壁垒不在于资讯本身，而在于**“筛选信息的品味”**和**“逻辑可视化的能力”**。通过极低的人力成本（AI 辅助 + 核心洞察），我们能为用户提供高溢价的决策辅助服务，这不仅仅是一个媒体，更是新一代个人投资者的外部大脑。

## 9. 业务流程优化与SOP (Business Process & SOP)

### 9.1 关键流程节点 (Key Nodes)

1. **选题与情报输入**：机构研报/宏观事件/行业数据进入待处理池。
2. **结构化拆解**：生成 `TL;DR`、核心观点、关键假设、风险点与可验证指标。
3. **内容编辑与校对**：统一术语、补齐数据来源、确保可读性与合规表达。
4. **付费墙策略**：确定免费预览与解锁条件（订阅/时间窗口）。
5. **工具化沉淀**：将内容要点映射为“蝴蝶效应图谱”节点，形成可复用知识结构。
6. **发布与分发**：站内发布 + 邮件/社媒分发 + SEO落地页。
7. **反馈与迭代**：收集用户反馈与行为数据，驱动选题与体验优化。

### 9.2 潜在瓶颈与优化方向 (Bottlenecks & Improvements)

- **选题过载**：建立“题库优先级评分”机制（热度、影响面、可操作性、差异化）。
- **内容生产不稳定**：采用模板化结构与检查清单，降低质量波动。
- **工具与内容割裂**：强制每篇深度解读产出一份最小可用图谱（MVP Graph），形成闭环。
- **转化链路不清晰**：定义关键触点与转化漏斗指标（曝光→阅读→登录→升级→留存）。

### 9.3 标准化业务操作流程 (SOP)

#### A. 内容生产SOP（每篇文章）

1. 选题登记：`slug`、来源机构、报告日期、标签、是否高级内容。
2. 内容产出：`summary_tldr` 与 `content_mdx` 完整生成并自检（逻辑、数据、引用）。
3. 图谱产出：至少包含 `root→event→impact→ticker` 的最小链路。
4. 发布前检查：格式/链接/敏感表述/付费墙生效/移动端阅读体验。
5. 发布后复盘：24小时内查看阅读与转化数据，补齐FAQ与图谱节点。

#### B. 反馈处理SOP（每日）

1. 分类：`bug/feature/billing/general`。
2. 评估：影响范围、复现概率、业务价值、实现成本。
3. 处理：高优先级问题进入迭代清单；订阅问题优先响应。
4. 复盘：每周汇总Top问题与Top需求，更新产品节奏。

## 10. 客户体验提升与闭环 (Customer Experience Loop)

### 10.1 客户旅程地图 (Journey Map)

| 阶段 | 用户目标 | 关键触点 | 主要阻力 | 指标建议 |
| :--- | :--- | :--- | :--- | :--- |
| 认知 | 判断是否值得关注 | 首页、精选文章 | 信息不够“可信/有用” | 首页点击率、停留时长 |
| 试用 | 快速获得价值 | TL;DR、预览内容 | 价值密度不足 | 文章阅读完成率 |
| 注册 | 保存进度/解锁能力 | 登录入口、魔术链接 | 登录成本 | 登录发起率、登录成功率 |
| 转化 | 获得完整内容与工具 | 付费墙/升级按钮、图谱工具 | 价格/权益不清 | 升级点击率、付费转化率 |
| 留存 | 持续获得可操作洞察 | 归档检索、图谱复用 | 新鲜度不足 | 7/30日留存、复访频次 |
| 推荐 | 分享给同圈层 | 分享按钮、导出 | 分享动机不足 | 分享率、邀请转化 |

### 10.2 反馈收集与分析机制 (Feedback System)

- 站内提供统一反馈入口，支持匿名或登录用户提交。
- 反馈内容结构化（类型、满意度、页面路径、用户信息可选），便于后续归因分析与优先级排序。
- 结合行为数据（关键触点事件），形成“定性反馈 + 定量漏斗”的双轮驱动。

### 10.3 满意度提升方案 (Satisfaction Plan)

1. **触点优化优先级**：优先优化高流量且高流失触点（付费墙、登录、图谱加载）。
2. **体验承诺**：明确响应时效（例如：订阅相关问题 24h 内响应；bug 72h 内给出结论）。
3. **快速迭代机制**：每周固定发布小版本，透明展示“本周已修复/已上线”。
4. **价值可视化**：在文章与工具中展示“你已经获得了什么”（节省时间、关键结论、可操作清单）。
